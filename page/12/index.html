如果有不清楚的，请详见README.rst文件.里面有详尽的安装配置指南。

目前对GIT仓库的目录管理均在/home/hw/Code/GitManager/gitosis-admin目录下，这本身是一个GIT仓库，因此对其中的修改需要进行正常的代码提交流程，等提交完毕以后，重启GIT用户下SSH的服务，即可生效。 目前该管理仓库的权限仅该账户持有，请谨慎修改。
1.进入目录gitosis-admin目录

2.仓库的建立，需要在git账户下，进入~/repositories目录,创建文件夹,初始化仓库
su git #切换账户
cd ~/repositories #进入存放代码仓库的根目录
mkdir test.git #创建文件夹
cd test.git #进入文件夹
git init –bare #对新建仓库进行git仓库初始化
exit #推出账户

3.添加新账户.将目标账户的id_rsa.pub文件拷贝到keydir目录下，修改文件名称符合*.pub即可，避免文件名冲突.

4.在gitosis.conf文件中添加仓库权限，注意名称对应，（由于对该仓库管理代码并不熟悉，目前发现添加的条目只能一对一的添加，因此目前比较繁琐。）
其中用户的名字对应的是*.pub的名字，即michael.pub 对应的名称为michael

5.提交修改
git add gitosis.conf
git commit
git push

6.切换GIT账户，重启SSH服务
su git
sudo /etc/init.d/ssh restart

Spigot server 安装

发表于 2019-03-14 | 分类于 配置

日期 2016-12-04

准备

  1. Windows系统
  2. Java 7环境
  3. Git 环境

开始安装

  1. 从以下网址下载BuildTools.jar.(目录路径尽量设为英文目录) 地址
  2. 在BuildTools.jar同级目录中打开git bash.
  3. 运行下面命令 java -jar BuildTools.jar –rev 1.10
  4. 等待下载安装完成

运行server

  1. 在server目录中创建一个start.bat文件，黏贴下面内容。内容中的spigot.jar改为目录中对应版本的spigot Jar包版本。等待安装完成。

    @echo off
java -Xms512M -Xmx1G -XX:MaxPermSize=128M -XX:+UseConcMarkSweepGC -jar spigot.jar
pause

2. 将server.properties 中 online-mode=false  关闭正版验证
  1. eula.txt 中 eula=true 启动

安装blocklyScript

  1. 打开blocklyScript地址。地址
  2. 找到BlocklyCraft Plugin Installation该行，进入对应的jar包的下载地址，下载blocklycraft.jar
  3. 下载完放入 server中plugins目录.
  4. 重新启动server.
